GET /api/v1/leads/

Lista próximos N leads não consumidos pelo cliente. Por padrão exclui processos já marcados como consumidos.

Query params
nivelPRECATORIO (default) · PRE_PRECATORIO · DIREITO_CREDITORIO
tribunalTRF1 · TRF3 · vazio = todos
limit1..10000 (default 5000)
min_score0..1 (default 0)
incluir_consumidos1/true/sim — retorna mesmo já consumidos

curl -H 'X-API-Key: <sua-chave>' \
  'https://voyager.was.dev.br/api/v1/leads/?nivel=PRECATORIO&limit=5000'

POST /api/v1/leads/consumed/

Marca processos como consumidos com resultado. Cada chamada cria registro novo (re-consumo permitido).

Body
{
  "consumos": [
    {"cnj": "1047957-32.2025.4.01.3300", "resultado": "validado"},
    {"cnj": "1036737-37.2025.4.01.3300", "resultado": "sem_expedicao"},
    {"cnj": "1093994-45.2024.4.01.3400", "resultado": "pago"}
  ]
}
Resultados aceitos
validado sem_expedicao erro pendente pago arquivado cedido
Exemplo
curl -X POST -H 'X-API-Key: <sua-chave>' \
  -H 'Content-Type: application/json' \
  -d '{"consumos":[{"cnj":"1047957-32.2025.4.01.3300","resultado":"validado"}]}' \
  'https://voyager.was.dev.br/api/v1/leads/consumed/'
Response (201)
{ "criados": 1, "nao_encontrados": [] }

Como a classificação funciona

Cada processo é avaliado por um modelo de Logistic Regression com 19 features (classe judicial, presença de movs específicas como precatório/RPV, contagem de movs, recência, idade, partes). O modelo retorna uma probabilidade [0,1] e a hierarquia abaixo determina a categoria.

CategoriaCritérioAção
💎 PRECATÓRIO score > 0.7 + tem expedição explícita (precatório/RPV no texto ou tipo) Fila imediata pra baixar autos
⏳ PRÉ-PRECATÓRIO score > 0.4 + classe de Cumprimento contra Fazenda Re-checar mensalmente
🌱 DIREITO CREDITÓRIO score > 0.2 + classe de Cumprimento Watch-list
não-lead demais ignorar

Triggers automáticos

  • Após cada sync (Datajud/DJEN), processo com mov nova é re-classificado automaticamente.
  • Cron 03:00 diário: re-classifica todos processos com mov nos últimos 7 dias (caso modelo mude ou features evoluam).
  • Re-treino: manual via novo registro em ClassificadorVersao (apenas 1 versão ativa por vez).
