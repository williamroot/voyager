# Voyager

## Contexto inicial obrigatório

No início de toda sessão neste projeto, leia a documentação em `.ia/` antes de responder ou agir. Esses arquivos contêm a visão geral, decisões de arquitetura, runbook operacional e padrões do projeto — sem eles, suas respostas ficam descontextualizadas.

Ordem sugerida de leitura:

1. `.ia/README.md` — índice da pasta
2. `.ia/OVERVIEW.md` — visão geral do sistema
3. `.ia/ARCHITECTURE.md` — arquitetura e componentes
4. `.ia/DATA_MODEL.md` — modelos e relações
5. `.ia/INGESTION.md` — pipeline de ingestão DJEN
6. `.ia/ENRICHMENT.md` — enrichers PJe
7. `.ia/DASHBOARD.md` — frontend HTMX/ECharts
8. `.ia/API.md` — REST API
9. `.ia/ACCOUNTS.md` — sistema de convites
10. `.ia/OPS.md` — runbook de produção (hosts, deploy, troubleshooting)
11. `.ia/PATTERNS.md` — padrões de código
12. `.ia/DECISIONS.md` — decisões técnicas com justificativa
13. `.ia/ROADMAP.md` — próximos passos

Consulte arquivos específicos sob demanda quando a tarefa pedir profundidade naquele tema.
